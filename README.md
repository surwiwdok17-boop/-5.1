Лабораторна робота №5: Розробка RESTful API
 Інформація про проєкт
Назва проєкту: Магазин “Хмарка”
Автори: [Дмитрук Дмитро. Харчук Віталій. Шуляк Артем. Павлик Олександр]
 Опис проєкту
Цей проєкт реалізує RESTful API для онлайн‑магазину, що дозволяє керувати товарами, клієнтами, замовленнями та відгуками. API підтримує CRUD‑операції, обробку помилок та інтеграцію з Swagger для тестування.
 Технології
Python 3.11
Flask
Flask-SQLAlchemy
Flasgger (Swagger UI)
SQLite
Flask-Admin
 Endpoints API
Отримати всі товари,
URL: /api/products
Метод: GET
Опис: Повертає список усіх товарів
Приклад відповіді:
json
{
  "status": "success",
  "data": [
    {
      "id": 1,
      "name": "Nova red bull",
      "price": 290,
      "image_url": "https://..."
    }
  ]
}
Створити новий товар,
URL: /api/products
Метод: POST
Опис: Додає новий товар до бази
Приклад запиту:
json
{
  "name": "Cola lemon",
  "price": 322,
  "image_url": "https://..."
}
json
{
  "status": "success",
  "message": "Product created"
}
Видалити товар,
URL: /api/products/<id>
Метод: DELETE
Опис: Видаляє товар за ID
Приклад відповіді:
json
{
  "status": "success",
  "message": "Product deleted"
}
Створити замовлення,
URL: /api/orders
Метод: POST
Опис: Створює нове замовлення з товарами та клієнтом
Приклад запиту:
json
{
  "client_id": 1,
  "product_ids": [1, 2]
}
Приклад відповіді:
json
{
  "status": "success",
  "order_id": 5
}
 Результати тестування в Swagger

Результат:

 Обробка помилок
400 Bad Request — якщо не передані обов’язкові поля (наприклад, name, price)
404 Not Found — якщо запитаний товар або замовлення не існує
500 Internal Server Error — якщо виникає помилка на сервері (наприклад, при з’єднанні з базою
